# CEX 跨所期现套利策略

本文记录第一版跨所期现套利自动化系统的策略边界、模拟盘规则和未来实盘差异。第一版只做模拟盘，不发送真实订单，但模型按真实跨所资金分布设计。

## 策略方向

### 买现货 + 空 U 本位永续

这是第一阶段主策略。系统在现货交易所买入现货，同时在永续合约交易所做空同等数量的 U 本位永续。

适用场景：

- 永续价格高于现货价格，存在正基差。
- 永续资金费率为正，空永续方预期收取资金费。
- 现货交易所有足够 USDT，合约交易所有足够保证金。

收益来源：

- 基差收敛收益。
- 资金费率收益。

主要风险：

- 合约保证金不足导致强平风险。
- 现货腿和合约腿无法同时成交，产生单边裸露。
- 资金费率转负，持仓从收钱变成付钱。
- 基差继续扩大，账面亏损触发风控。

### 卖出现货库存 + 多 U 本位永续

这是第二阶段策略，只允许使用已有现货库存，不做借币卖空、不做现货杠杆。

适用场景：

- 现货价格高于永续价格，存在负基差或反向基差。
- 永续资金费率为负，多永续方预期收取资金费。
- 现货交易所已经持有足够现货库存，合约交易所有足够保证金。

收益来源：

- 反向基差收敛收益。
- 负资金费率下，多永续获得的资金费。

主要限制：

- 系统不能为了执行该策略去借币。
- 现货库存不足时必须拒绝机会。
- 如果未来接真实账户余额，仍要保留手动模拟库存能力，方便模拟盘复盘和压力测试。

## 收益模型

统一使用以下公式判断机会：

```text
预期净收益 = 基差收益 + 预计资金费率收益 - 手续费 - 滑点 - 安全缓冲
```

字段说明：

- 基差收益：现货成交价和永续成交价之间的价差贡献。
- 预计资金费率收益：按当前资金费率和预计结算次数估算。
- 手续费：交易所明确收取的费用，按现货 taker 和合约 taker 费率分别计算。
- 滑点：不是手续费，而是盘口深度不足或价格移动导致的成交价偏移成本。
- 安全缓冲：防止模拟盘过度贴近理论收益，为行情延迟、撮合延迟和估算误差留余量。

第一版滑点使用固定比例估算，后续可升级为订单簿深度撮合。

## 模拟盘和真实盘区别

模拟盘：

- 成交价来自盘口估算。
- 买入使用 ask，卖出使用 bid。
- 手续费、滑点和安全缓冲按配置扣减。
- 每个交易所独立维护虚拟 USDT、现货余额、合约保证金、冻结资金和持仓。
- 模拟盘用于验证机会质量、收益核算、资金分布、风控和清仓流程。

真实盘：

- 成交价必须来自交易所订单成交回报。
- 收益必须使用真实成交均价、成交数量、真实手续费、真实资金费率和实际平仓结果。
- 真实盘必须处理部分成交、撤单失败、API 超时、重复回报、合约持仓模式和保证金模式。
- 真实收益可能显著偏离模拟盘，模拟盘结果不能直接当作真实收益。

## 资金模型

第一版使用每个交易所独立虚拟账户：

```text
Binance: USDT、现货余额、合约保证金、合约持仓
OKX:     USDT、现货余额、合约保证金、合约持仓
Bitget:  USDT、现货余额、合约保证金、合约持仓
```

这样做是因为跨所套利不能假设资金能瞬间跨交易所移动。机会执行前必须检查实际涉及的现货所和合约所资金是否足够。

## 熔断和清仓

熔断含义：

```text
停止开新仓 + 对已有套利组合生成紧急平仓动作
```

触发条件：

- 单笔亏损超过阈值。
- 日亏损超过阈值。
- 单所余额不足或保证金不足。
- 行情延迟或数据失真。
- 单腿成交失败。
- 交易所 API 异常。

清仓原则：

- 清仓不能盲目市价全平。
- 优先保持两腿对冲关系，避免只平一边制造更大的裸露风险。
- 如果单腿失败，只成交的一边要立即反向平掉。
- 如果两边成交数量不一致，保留较小数量的对冲部分，剩余裸露部分优先平掉。
- 如果某个交易所不可访问，系统应进入紧急模式并提示人工介入。

## 第一版实现边界

- 覆盖 Binance、OKX、Bitget。
- 支持任意现货交易所 x 任意 U 本位永续合约交易所组合，包括同所期现。
- 交易对使用自定义白名单，默认可先用 BTCUSDT、ETHUSDT、SOLUSDT。
- 第一版只做模拟盘，不真实下单。
- 反向策略只使用已有现货库存，不借币、不做现货杠杆。
